Chapter 4 - User Profile Managment
 
The users are divided into following Batch Orchestrator Groups:
  • BO Process Engineer
  • BO Production Supervisor
  • BO Process Control Engineer
  • BO Process Operator
  • BO Product Admin
NOTE: For information on adding the BO users manually to the Orchetrator (mandatory), please refer to the Batch Orchestrator Installation and Configuration Guide.
 
These Batch Orchestrator User Groups, once added to the Experion, will have certain sets of roles to perform. Based on these different user groups, User Profile Management will allow the Product Admin to define the roles allowed for a particular user to act:




For example, as shown above a Product Engineer may have the permission to create Formula Sets but not to approve them and on the other hand, Production Supervisor may have the role to approve the Formula Sets but not to create them. This way Product Admin can manage the tasks and actions among the Users, which will send out certain action notification only to the required group of users.
 
Following table summarises the action roles and tasks of different User Groups.
 
Tasks \ User Groups Product Admin Product Engineer Product Control Engineer Process Operator Production Supervisor
Create Formula Sets yes yes no no no
Modify Formula Sets yes yes no no no
Delete Formula Sets yes no no no no
View Audit Trail yes no no yes yes
Compare Formulas yes no no yes yes
Approve/Reject Formula yes no no no yes
Update Permission yes no no no no
View Formula/Formula Sets yes yes yes yes yes
Create Batch yes no no yes yes
 
 
 
Product Admin can update permission, and allow access to the listed roles to User Groups in Batch Orchestrator as follows:
  1. Select a role in the User Group Action or View catagory




  2. Select the "Permission Allowd" or "Permission Denied" icon, as per requirment.

  3. System Assigned Permission Allowed / System Assigned Permission Denied
 
Procedure to perform above roles (action / view) are discussed further in the chapter ahead.
